Policy Notes

  • 'strict-dynamic' in script-src allows trusted scripts to load additional scripts dynamically — maintaining security while enabling legitimate CDN-loaded dependencies
  • The nonce P3rf0rm4lyt1c is applied to all inline <script> and <style> blocks, preventing injection of unauthorized inline code
  • 'unsafe-inline' in style-src on main pages is a fallback for browsers that don't support strict-dynamic in style directives
  • upgrade-insecure-requests forces all HTTP requests to HTTPS across the entire page
  • Blog article pages and the 404 page have a more restrictive CSP — no Tawk.to, no unsafe-inline
  • The 404 page has no script-src directive — scripts are blocked entirely
  • form-action is restricted to 'self' and https://docs.google.com, preventing form data exfiltration to unknown endpoints
  • object-src 'none' blocks all plugin content (Flash, Java, etc.)
